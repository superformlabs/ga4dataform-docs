What's compilation in Dataform

What is Compilation in General?​

In computer science, compilation is the process of translating code written in a high-level language (like Java, C, or TypeScript) into a lower-level representation (machine code, bytecode, or JavaScript) that can actually be executed. The compiler ensures that the code is syntactically valid, resolves dependencies, and produces an executable artifact. Without compilation, your code is just text — understandable by humans but not directly executable by a machine.

Compilation in Dataform​

Dataform applies the same principle to data pipelines. Instead of writing raw SQL, you use SQLX, a higher-level abstraction that includes:

  • ref() statements to declare dependencies.
  • Configurations such as type: "incremental".
  • Macros and includes to avoid repetitive code.

These constructs don’t exist in your data warehouse’s SQL dialect. Before execution, Dataform needs to compile them into plain SQL that your warehouse can run.

During compilation, Dataform:

  • Resolves references (turning ref("orders") into the fully qualified table name).
  • Validates configurations and dependencies.
  • Builds a directed acyclic graph (DAG) of transformations.
  • Generates the final SQL queries and execution plan.

The output is a set of compiled SQL statements and a dependency graph that ensures the right execution order.

Why You Need to Compile Before Running Workflows​

When you push a new commit to your Dataform project, the code in your repository changes. However, your workflows don’t automatically pick up these changes unless you recompile. Why?

Workflows run against the compiled graph, not directly against your repository code.

If you don’t recompile, your workflow may still point to the old compiled manifest, meaning your latest changes aren’t reflected.

This can cause confusion: your repo shows updated SQLX, but your scheduled or triggered workflow still runs the old version.

In other words, compilation acts as a bridge between your repository and your warehouse execution. Until you update that bridge by compiling, the workflow will not see your new code.

Key Takeaway​

Compilation in Dataform is the step that transforms your human-friendly SQLX into machine-executable SQL for your data warehouse. Just like in software engineering, skipping compilation means your latest code changes never make it into production. So whenever you commit new transformations, remember: compile first, then run.
